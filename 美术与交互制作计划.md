# 岛屿保卫战：美术与交互制作计划

## 1. 已确定方向

### 1.1 产品定位

《岛屿保卫战》是一款儿童向轻操作塔防游戏，目标体验是：

- 4-8 岁孩子能快速上手。
- 家长能接受其中的学习设计。
- 画面不再像网页草稿，而像一款精致、完整、有统一风格的小型 Steam 游戏。

核心描述：

> 精致玩具模型风的等距小岛塔防。玩家作为“小岛队长”，带领小精灵、建造炮台、守护星星，用奖励型学习补给获得成长。

### 1.2 十问结论

| 项目 | 结论 |
| --- | --- |
| 整体美术 | 精致玩具模型风 |
| 镜头视角 | 稍微倾斜的等距视角 |
| 主色调 | 温暖童话自然色 |
| 怪物气质 | 可爱但有一点捣蛋威胁 |
| 玩家操作 | 轻操作带队战斗 |
| 学习系统 | 奖励型学习补给 |
| 炮台复杂度 | 少量炮台，每个差异明显 |
| 主角方向 | 小岛队长 / 小小守护者 |
| 战斗反馈 | 清楚爽快但不吵 |
| 节奏结构 | 短关卡 + 明确成长奖励 |

## 2. 体验支柱

### 2.1 守岛主角感

玩家不是旁观者，而是岛上的小队长：

- 可以移动到关键位置。
- 可以带小精灵小队。
- 可以释放闪电、星光等技能。
- 可以在紧急时刻亲自拦截怪物。

炮台仍然是防守主力，玩家操作负责“救场”和“带队”。

### 2.2 一眼看懂

每个游戏元素都要有强识别：

- 怪物类型从轮廓、动作、颜色上能区分。
- 炮台功能从造型和攻击特效上能区分。
- 可建造位置、路径、星星核心、糖果、小精灵都不能互相混淆。

### 2.3 学习是奖励，不是惩罚

学习题目改为“星星补给”或“小精灵训练”：

- 答对题目给星星、护盾、免费升级、召唤小精灵。
- 失败复活可以保留，但不作为主要学习入口。
- 学习入口应表现为礼物、宝箱、补给船、精灵课堂，而不是处罚弹窗。

### 2.4 短关卡强反馈

每关目标时长 2-3 分钟：

- 开局快速建塔。
- 中段出现压力点。
- 结尾出现小高潮。
- 结算给清楚奖励和下一关期待。

## 3. 美术风格规范

### 3.1 总体视觉关键词

- 精致玩具
- 等距小模型
- 温暖童话
- 自然材质
- 糖果点缀
- 软萌角色
- 清晰战斗反馈

避免：

- 纯网页扁平图形
- 高饱和手游广告感
- 恐怖怪物
- 过多紫蓝霓虹
- 复杂写实材质
- UI 像后台管理系统

### 3.2 镜头

目标镜头：

- 倾斜等距视角，类似看一个桌面玩具岛。
- 地图不完全俯视，地面、炮台、角色都有可见侧面。
- 路线和建造点仍要清楚，不牺牲塔防可读性。

技术实现建议：

- 当前 Canvas 可以先做“伪等距”：把角色和物体绘制成有侧面、椭圆阴影、顶部高光的 2.5D 资产。
- 第二阶段再抽象坐标转换：世界坐标保留 2D，渲染层做轻微 y 轴压缩和层级排序。

### 3.3 色彩

主色建议：

- 海水：柔和蓝绿，不刺眼。
- 草地：温暖绿，避免荧光绿。
- 沙滩：奶油黄、浅焦糖。
- 木头：蜂蜜棕、浅木色。
- 星星：暖金色。
- 糖果点缀：粉色、薄荷绿、淡紫、彩虹，但只做强调色。

比例建议：

- 60% 自然色：海水、草地、沙滩、木头。
- 25% 游戏功能色：星星、血条、路径、塔台。
- 15% 糖果和特效色：粉、紫、彩虹、电光。

### 3.4 材质

所有物体像玩具模型：

- 草地：软胶或绒面质感。
- 沙滩：奶油糖霜质感。
- 炮台：木头、糖果、宝石、小金属件。
- 小精灵：半透明软糖或发光精灵。
- 星星核心：金色水晶/星星徽章。
- 怪物：软胶手办，不黏腻、不恐怖。

### 3.5 轮廓和比例

角色比例：

- 大头、小身体、短四肢。
- 眼睛大，表情明确。
- 轮廓圆润，少尖刺。

怪物比例：

- 圆圆怪：圆球主体，短脚，调皮表情。
- 盾盾怪：前方有蓝色护盾弧面。
- 乌龟怪：横向宽，壳厚，走得慢。
- 蝙蝠怪：翅膀大，身体小，移动快。
- 叛变精灵：和小精灵相似，但颜色偏暖橙，表情顽皮。
- 月球 Boss：大圆月身体，表面坑洞，缓慢但压迫感强。

### 3.6 UI 风格

UI 像玩具按钮和冒险仪表：

- 顶部资源条：像木质/玻璃质感小牌。
- 底部炮台栏：像玩具盒托盘。
- 技能按钮：大图标、冷却数字清楚。
- 结算界面：像星星奖章板。
- 学习界面：像补给宝箱/精灵训练卡。

避免：

- 普通网页卡片感。
- 大段说明文字。
- 过多透明毛玻璃。
- 按钮文字过长。

## 4. 角色与素材计划

### 4.1 主角：小岛队长

来源：

- 使用项目文件夹中给出的照片作为人物参考。
- 不直接使用真实照片进入游戏。
- 用 image gen 2 生成统一风格的卡通素材。

主角设定：

- 8 岁左右小朋友形象。
- 蓝白守岛服。
- 小披肩或队长领巾。
- 手持星星指挥棒。
- 表情勇敢、专注、可爱。

需要生成的主角素材：

| 素材 | 用途 |
| --- | --- |
| 主角站立正面/等距角 | 游戏内默认状态 |
| 主角跑步 4 帧 | 移动动画 |
| 主角挥动星星棒 3 帧 | 自动攻击 |
| 主角释放技能姿势 | 闪电/星光技能 |
| 主角头像 | HUD、菜单、结算 |
| 主角胜利动作 | 通关结算 |

### 4.2 小精灵

方向：

- 糖果被打爆后释放的小伙伴。
- 半透明、发光、像软糖和星星结合。
- 跟随主角形成小队。

需要素材：

- 粉色精灵：普通攻击。
- 绿色精灵：恢复/治疗。
- 黄色精灵：星星补给。
- 蓝色精灵：护盾。

### 4.3 炮台

保持 4 种主炮台：

| 炮台 | 视觉 | 功能 |
| --- | --- | --- |
| 电炮台 | 蓝色电池/小线圈 | 快速射击 |
| 寒冰塔 | 冰晶、蓝白雪花 | 减速 |
| 饼干炮 | 饼干炮筒、奶油底座 | 范围爆炸 |
| 彩虹镜 | 彩虹棱镜、宝石反射器 | 弹射攻击 |

每个炮台需要：

- 1-5 级外观变化。
- 底座。
- 开火动画。
- 子弹/命中特效。

### 4.4 怪物

怪物要可爱但捣蛋：

- 表情偏“我要偷星星”，不是恐怖。
- 被打时有夸张反应。
- 死亡表现是弹开/变星星碎片，不血腥。

需要素材：

| 怪物 | 素材要求 |
| --- | --- |
| 圆圆怪 | 走路/受击/消失 |
| 盾盾怪 | 护盾满/护盾破裂/受击 |
| 乌龟怪 | 慢走/缩壳/受击 |
| 蝙蝠怪 | 飞行翅膀动画/受击 |
| 叛变精灵 | 远程攻击/恢复正常可选 |
| 月球 Boss | 移动/冲击/受击/低血量 |

## 5. 交互设计计划

### 5.1 核心循环

单关循环：

1. 进入关卡。
2. 展示路线和怪物预告。
3. 玩家用初始星星建炮台。
4. 怪物分波进攻。
5. 糖果被打爆，生成小精灵。
6. 玩家带小精灵救场。
7. 触发学习补给，获得奖励。
8. 击退最后一波。
9. 结算奖励，解锁下一关或强化内容。

### 5.2 玩家操作

桌面端：

- 鼠标点击移动。
- 点击建造点建炮台。
- 点击炮台升级/回收。
- 技能按钮释放技能。
- WASD/方向键移动作为辅助。

移动端：

- 点击地图移动。
- 点击建造点弹出炮台选择。
- 技能按钮固定在右下。
- 炮台栏横向滑动。

### 5.3 建造交互

目标体验：

- 可建造点要像岛上的小圆台。
- 鼠标悬停或触摸时发光。
- 星星不足时给轻微震动/红色提示。
- 建造成功时有星星光效和清脆音效。

改进项：

- 先选建造点，再弹出炮台轮盘。
- 或底部选炮台，再点建造点建造。
- 儿童模式建议默认“点建造点弹选择”，减少误操作。

### 5.4 升级交互

升级不只是数字变化：

- 炮台模型变高/更亮。
- 炮筒变大。
- 底座加星星徽章。
- 攻击特效更明显。

满级反馈：

- 显示 5 颗小星。
- 炮台周围有轻微常驻光圈。

### 5.5 学习补给交互

新系统方向：

- 每关中出现 1-2 次“补给机会”。
- 形式可以是补给船靠岸、星星宝箱、精灵训练卡。
- 玩家主动点击补给，进入短题目。

答对奖励池：

- +60 星星。
- 免费升级一次最低等级炮台。
- 星星核心获得 2 次护盾。
- 召唤 1 个小精灵。
- 冻结全场 3 秒。

答错处理：

- 不惩罚。
- 给较小奖励，例如 +15 星星。
- 显示正确答案。
- 鼓励语气保持轻松。

### 5.6 战斗反馈

目标：

- 清楚爽快但不吵。
- 特效不遮挡怪物和路径。
- 数字反馈少而关键。

规则：

- 小伤害不一直跳数字。
- 暴击、技能、击杀、奖励才显示大数字。
- 冰冻要有蓝色减速圈。
- 闪电要有 3x3 区域提示。
- Boss 低血量要有阶段变化。

## 6. 关卡与节奏计划

### 6.1 10 关结构

| 关卡 | 场景 | 新内容 |
| --- | --- | --- |
| 1 | 糖果海湾 | 电炮台、圆圆怪 |
| 2 | 糖果海湾 | 蝙蝠怪 |
| 3 | 糖果海湾 | 糖果变小精灵 |
| 4 | 焦糖沙地 | 寒冰塔强化、乌龟怪 |
| 5 | 焦糖沙地 | 月球小 Boss |
| 6 | 焦糖沙地 | 饼干炮、盾盾怪组合 |
| 7 | 泡泡水门 | 彩虹镜、叛变精灵 |
| 8 | 泡泡水门 | 多路压力 |
| 9 | 泡泡水门 | 学习补给高价值奖励 |
| 10 | 月光水门 | 潮汐月王最终 Boss |

### 6.2 每关时长

目标：

- 第 1-3 关：90-120 秒。
- 第 4-7 关：120-150 秒。
- 第 8-10 关：150-180 秒。

### 6.3 难度原则

- 失败原因要清楚。
- 失败后给复盘提示。
- 失败不能让孩子卡太久。
- 学习补给可以作为软性难度调节。

## 7. 当前工程改造计划

### 7.1 第一阶段：风格规范落地

目标：

- 当前画面从草稿风改成统一的正式风格。
- 保持现有玩法不大改。

任务：

- 统一颜色配置。
- 把所有程序化图形改成 2.5D 玩具风。
- 优化 HUD、炮台栏、结算界面。
- 增强建造、升级、技能反馈。

验收：

- 截图看起来像完整游戏，而不是 Canvas 原型。
- 玩家、怪物、炮台、路径、UI 属于同一个世界。

### 7.2 第二阶段：主角和关键素材生成

目标：

- 用 image gen 2 基于照片生成主角卡通素材。
- 替换当前程序化主角。

任务：

- 制作主角形象提示词。
- 生成主角头像、站立、跑步、攻击、技能姿势。
- 生成小精灵、怪物、炮台关键资产。
- 统一透明背景和尺寸规范。

素材规范：

- 角色单体：512x512 PNG。
- UI 头像：512x512 PNG。
- 炮台：512x512 PNG。
- 怪物：512x512 PNG。
- 图标：256x256 PNG。

命名建议：

- `assets/characters/captain_idle.png`
- `assets/characters/captain_run_01.png`
- `assets/allies/sprite_pink_idle.png`
- `assets/enemies/blob_idle.png`
- `assets/towers/spark_lv1.png`
- `assets/ui/captain_portrait.png`

### 7.3 第三阶段：等距渲染改造

目标：

- 从俯视图过渡到倾斜等距视角。

任务：

- 建立世界坐标和屏幕坐标转换。
- 所有实体按 y 坐标排序绘制，保证遮挡正确。
- 路径改成等距地面块风格。
- 建造点改成凸起平台。
- 角色脚下阴影统一。

验收：

- 地图有立体感。
- 路线仍然清楚。
- 操作点击不混乱。

### 7.4 第四阶段：学习系统重做

目标：

- 从失败惩罚改成奖励补给。

任务：

- 增加补给宝箱/补给船机制。
- 关卡中按节奏触发学习机会。
- 答题奖励接入星星、护盾、免费升级、小精灵。
- 保留失败复活答题作为次要机制。

验收：

- 孩子主动愿意点补给。
- 答题不会打断核心战斗太久。
- 家长能看出学习价值。

### 7.5 第五阶段：手感与反馈打磨

目标：

- 每个操作都有清楚反馈。

任务：

- 建造音效、升级音效、技能音效。
- 怪物受击动画。
- 炮台开火节奏。
- Boss 出场演出。
- 通关奖励动画。

验收：

- 玩家不看说明也知道发生了什么。
- 战斗爽但屏幕不乱。

## 8. Image Gen 2 素材生产流程

### 8.1 输入材料

使用项目中的人物照片作为参考：

- `小岛.jpeg`
- `小岛和爸爸.jpeg`

用途：

- 参考主角脸型、气质、年龄感。
- 不用于直接贴图。
- 输出统一游戏风格的卡通角色。

### 8.2 主角提示词方向

核心提示词应包含：

- 精致玩具模型风。
- 儿童守岛队长。
- 蓝白守岛服。
- 星星指挥棒。
- 温暖童话自然色。
- 等距游戏角色。
- 透明背景或纯色背景便于抠图。
- 保持参考照片中小朋友的整体气质。

避免：

- 写实照片。
- 恐怖表情。
- 过度日漫。
- 太像手游广告。
- 复杂背景。

### 8.3 资产验收标准

每张素材检查：

- 是否统一风格。
- 是否适合 4-8 岁。
- 是否能在小尺寸下识别。
- 是否能放进等距场景。
- 是否有干净边缘。
- 是否避免真实照片感。

正式美术第一版完成时，应满足：

- [ ] 主角不是程序化小人，而是统一风格卡通角色。
- [ ] 炮台至少 4 种有清楚差异。
- [ ] 怪物至少 5 种有清楚差异。
- [ ] 小岛场景有立体感。
- [ ] UI 不像网页控件。
- [ ] 学习题目是奖励补给入口。
- [ ] 每关 2-3 分钟。
- [ ] 失败后不产生挫败感。
- [ ] 截图能看出“精致玩具模型风”。
- [ ] 8 岁孩子不看说明也能玩第一关。
